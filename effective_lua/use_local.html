# Use local 使用 local 在运行任何代码之前，Lua将源转换（预编译）为内部格式。此格式是虚拟机的一系列指令，类似于真实CPU的机器代码。然后通过C代码解释此内部格式，c代码本质上是一个带着很大的switch的while循环，每个case用于每条指令。也许你在某个地方已经了解过，Lua自5.0版以来使用了基于寄存器的虚拟机。这个虚拟机的“寄存器”不是对应于CPU中的实际寄存器（对应实际寄存器就会不可移植，而且实际可用的寄存器数量非常有限），而是使用堆栈（实现为数组加上一些索引）来实现它的寄存器。 每个活动函数都有一个激活记录，它是一个堆栈 片段，其中函数存储其寄存器。所以，每个功能都有自己的功能 registers2 。每个函数最多可使用250个寄存器，因为每条指令 只有8位来指代寄存器。 鉴于寄存器数量众多，Lua预编译器能够存储所有寄存器 寄存器中的局部变量。结果是访问局部变量非常 在Lua很快。例如，如果a和b是局部变量，那么Lua语句就像 a = a + b生成一条指令：ADD 0 0 1（假设a和b 分别在寄存器0和1中）。为了比较，如果a和b都是 全局，添加的代码将是这样的：
